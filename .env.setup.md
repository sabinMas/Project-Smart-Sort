## Step 1: SendGrid Setup (10 minutes)

SendGrid handles email ingestion. Here's how to set it up:

### 1.1 Create SendGrid Account

1. Go to: https://sendgrid.com
2. Click "Sign Up" → Choose "Start Free"
3. Fill in registration form
4. Verify your email address
5. Create account

### 1.2 Get API Key

1. From dashboard, go to **Settings → API Keys**
2. Click "Create API Key"
3. Name it: `hackathon-inbound`
4. Select "Restricted Access"
5. Enable only: **Mail Send** and **Inbound Parse**
6. Click "Create & View"
7. **Copy the API key** (save it immediately!)

```env
SENDGRID_API_KEY=SG_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### 1.3 Set Up Inbound Parse Webhook

⚠️ **Do this AFTER you have ngrok running (Step 7)**

1. From dashboard, go to **Settings → Inbound Parse**
2. Click "Create New Webhook"
3. **Hostname:** `system@yourdomain.com` (or use your test email)
4. **URL:** `https://your-ngrok-url.ngrok.app/webhooks/email`
5. **Post the raw, full MIME message** - Enable this
6. Save

### 1.4 Get Verify Token

SendGrid sends a verification header with webhooks. You need to extract this token:

1. Go to **Settings → Inbound Parse**
2. Click on your webhook
3. The verify token is shown as "HTTP POST URL"
4. The token is in the URL: `?verify=TOKEN_HERE`

```env
SENDGRID_VERIFY_TOKEN=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### 1.5 Test

```bash

```

## Step 2: Box Setup (20 minutes)

Box handles file management and integration.

### 2.1 Create Box Developer Account

1. Go to: https://developer.box.com/console
2. Click "Create New App"
3. Select **Custom App** → **Server Authentication** → **OAuth 2.0 with JWT**
4. Name it: `SmartInbox` or `Hackathon-App`
5. Click "Create App"

### 2.2 Get API Credentials

1. Go to your app's "Configuration" tab
2. Under **OAuth 2.0 Credentials**, you'll see:

```env
BOX_CLIENT_ID=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
BOX_CLIENT_SECRET=yyyyyyyyyyyyyyyyyyyyyyyyyyyyyy
```

### 2.3 Get Enterprise ID

⚠️ The enterprise ID is different based on your account type

**For Free Developer Account:**
1. Go to https://account.box.com/settings/general
2. Look for "Account ID" or "Enterprise ID"
3. Format: `free-developer-account-youremail-com`

```env
BOX_ENTERPRISE_ID=free-developer-account-masonsabin-gmail-com
```

### 2.4 Generate Developer Token (for testing)

1. In your app's Configuration tab
2. Scroll to "App Tokens"
3. Click "Generate Token"
4. Copy the **Primary Access Token**

```env
BOX_DEVELOPER_TOKEN=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

⚠️ **Note:** Developer tokens expire in 1 hour. For production, implement OAuth flow.

### 2.5 Add Webhook Keys

1. Go to your app's Configuration tab
2. Under **Webhook Configuration**, set:

```env
WEBHOOK_PRIMARY_KEY=your-primary-key-here
WEBHOOK_SECONDARY_KEY=your-secondary-key-here
```

Can be any random strings for testing. In the code:
```python
WEBHOOK_KEYS = {
    "primary": "your-primary-key-here",
    "secondary": "your-secondary-key-here"
}
```

### 2.6 Test Box Connection

```bash
# Run this after updating .env
python -c "
from box_sdk_gen.client import BoxClient
from box_sdk_gen.schemas import BoxDeveloperTokenAuth

auth = BoxDeveloperTokenAuth(token='YOUR_TOKEN_HERE')
client = BoxClient(auth=auth)
user = client.users.get_current_user()
print(f'✅ Connected as: {user.name}')
"
```

## Step 3: LLM Provider Setup (10 minutes)

You need an LLM to classify documents. Choose one:

### Option A: Cerebras (Recommended)

1. Go to: https://console.cerebras.ai
2. Click "Sign Up" (free tier available)
3. Create account and verify email
4. Go to **Dashboard → API Keys**
5. Click "Create API Key"
6. Name it: `hackathon`
7. Copy the key

```env
LLM_PROVIDER=cerebras
CEREBRAS_API_KEY=csk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### Option B: Groq (Backup Option)

1. Go to: https://console.groq.com
2. Sign up (free tier: 30 requests/minute)
3. Go to **API Keys**
4. Create key: `hackathon`
5. Copy the key

```env
GROQ_API_KEY=gsk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### Option C: Google Gemini

1. Go to: https://ai.google.dev
2. Click "Get API Key"
3. Create new API key in Google Cloud Console
4. Copy the key


**Recommendation:** Start with **Cerebras** (fastest for hackathon), have **Groq** as backup.

## Step 4: ngrok Setup (5 minutes)

ngrok exposes your local server to SendGrid webhooks.

### 4.1 Download ngrok

1. Go to: https://ngrok.com/download
2. Download for your OS (Windows/Mac/Linux)
3. Unzip to a folder
4. Add to PATH (optional, for easy access)

### 4.2 Start ngrok Tunnel

```bash
# Start tunnel on port 8000 (where FastAPI runs)
ngrok http 8000

# You'll see output like:
# ngrok
# Session Status    online
# Session Expires   1:59:47 AM
# Update Available  (version 3.x.x, Ctrl-U to update)
#
# Forwarding        https://abc123xyz456.ngrok.app -> http://localhost:8000
```

**Save the URL:** `https://abc123xyz456.ngrok.app`

```env
SENDGRID_INBOUND_URL=https://abc123xyz456.ngrok.app/webhooks/email
```

⚠️ **Important:** 
- URL changes every time you restart ngrok
- Keep ngrok running during development
- Update SendGrid webhook URL if ngrok restarts

## Troubleshooting

### "ModuleNotFoundError: No module named 'backend'"

```bash
# Ensure you're in the project root
cd /path/to/hackathon-skeleton

# Verify structure
ls -la backend/
# Should show: __init__.py, main.py, domain_1_email, etc.
```

### "Webhook not received"

1. ✅ ngrok tunnel running? (`ngrok http 8000`)
2. ✅ URL updated in SendGrid? (includes ngrok URL)
3. ✅ Email sent to `system@yourdomain.com`?
4. ✅ Check ngrok logs for incoming requests
5. ✅ Check backend logs for errors

### "Classification slow or failing"

1. Check LLM provider status
2. Verify API key is correct
3. Check internet connection
4. Try backup provider (Groq if Cerebras failing)
5. Look at backend logs for error message

### "Box file not appearing"

1. Check Box credentials (BOX_CLIENT_ID, BOX_CLIENT_SECRET)
2. Verify Enterprise ID is correct
3. Check backend logs for Box API errors
4. Ensure developer token hasn't expired
5. Check Box folder structure exists

### "Tests failing"

```bash
# Run specific test with verbose output
pytest backend/domain_1_email/tests/test_routes.py -vv -s

# Check what's wrong
# Usually missing .env or API keys
```

## Security Checklist

Before going live:

- ✅ Never commit .env to git
- ✅ Rotate all API keys before production
- ✅ Use environment-specific credentials
- ✅ Implement proper OAuth (not just dev tokens)
- ✅ Add rate limiting to endpoints
- ✅ Add request validation and sanitization
- ✅ Set up HTTPS (not just HTTP)
- ✅ Add request signing for webhooks
- ✅ Use secrets management (not files)
